# 바크로 월별 매출 보고서 — 기획안 v3.0

## 0. 한 줄 요약
`sales.csv` (매출 raw) + `target.csv` (월별 목표)를 입력으로 받아 **매월 임원 보고용 다각도 분석 대시보드**를 생성하는 Next.js + ECharts + shadcn 로컬 웹 앱. URL 쿼리(`?month=2026-04`)로 보고월을 바꿔 동일 규격을 매달 재사용. **2026-04를 기준월**로 첫 빌드.

### v3 vs v2 핵심 차이

- **탭 10개** (v2: 9개) — 신규 **거래처 분석** 탭 추가, **인사이트 → 심층 분석** 으로 재포지셔닝.
- **거래처/딜러 심층 분석** — 동면 복귀 / 분기 절벽 / 상실된 핵심 거래처 / 신규 진입 자동 감지. 거래처 1곳 또는 2곳 비교. B2B 탭에 영업사원 6개월 sparkline + 신규/이탈 거래처.
- **모든 탭 상단에 자동 인사이트** — 휴리스틱 기반 한국어 불릿 (예: "네이버 스마트스토어 +150%", "롯데면세점 -32%"). LLM 없음.
- **사람 코멘트는 선택** — `/insights` 페이지 하단에만 노출, 없으면 자동 분석만.
- **구조적 성능** — `lib/facts.ts` 가 CSV 로드 시 모든 차원 사전 집계(팩트 큐브). 모든 라우트에 `loading.tsx` (탭 전환 즉시 스켈레톤).

### v2 vs v1 차이 (참고)

- 9개 탭 (종합·목표 달성·수출·B2B·B2C·면세점·브랜드 분석·변동 분석·인사이트)
- target.csv 통합으로 이번달/분기 목표 달성률을 핵심지표 카드와 매트릭스에 노출
- 모든 라벨 한국어 (영어 약자 제거), 비교는 절대값+차이금액+변화율 동시 표시
- "왜 늘었나/줄었나" 변화 요인 워터폴 차트 + 신규/이탈 거래처 + 신제품 효과 분석

## 1. 기술 스택과 실행 방식

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Next.js 15 (App Router)** | Server Component로 CSV 집계를 요청 시점에 끝내고, 클라이언트는 차트만 렌더 |
| 차트 | **Apache ECharts + echarts-for-react** | 워터폴/트리맵/히트맵/게이지 등 풍부, 한국어 툴팁/폰트, Apache 2.0 무료 |
| UI | **shadcn/ui (Radix) + Tailwind 3** | 카드/탭/배지/셀렉트/툴팁 폴리시 |
| 표 | **TanStack Table** | 정렬/검색/페이지/sticky 헤더 |
| 데이터 처리 | Node 단에서 **PapaParse + 자체 집계 + 팩트 큐브** | 외부 의존 최소, sales.csv를 한 번 파싱하고 모든 차원을 사전 집계 → 모든 화면 공유 |
| 실행 | `npm run dev` → http://localhost:3000 | 매달: `sales.csv` + `target.csv` 교체 → 새로고침 |
| 스냅샷 | `npm run snapshot -- 2026-04` → 단일 HTML | 임원 회람용 |

## 2. 데이터 파이프라인

### 2.1 원본 컬럼 (17개)
`채널, 날짜, 주문번호, 제품명, 품목코드, 판매수량, 실 매출, 주문금액, 할인금액, 수수료, 배송비, 정산금액, 딜러, 거래처, 거래처 사업형태, 원가, 브랜드`

### 2.2 정제 규칙

| 이슈 | 처리 |
|---|---|
| 날짜 포맷 혼재 (`2023-12-21`, `2026. 4. 9`) | 정규식으로 분리 → ISO 변환 |
| 숫자 콤마/공백 (`11,400`) | 콤마 제거 후 Number |
| 원가 `#N/A` | NaN 처리, 마진 KPI에서 제외하고 데이터 품질 배지로 비율 노출 |
| 실매출 0 행 | **매출 집계에서 제외**, 별도 "비매출 출고" 카드/탭으로 분리 노출 |
| 주문번호 단위 집계 | 주문건수(unique 주문번호), AOV(주문당 평균) 계산 |
| 빈 딜러 (203행) | "미지정"으로 통일 |

### 2.3 파생 컬럼 (앱 내부에서만)

- `category`: `수출` | `B2B` | `B2C` | `면세점`
- `b2bCustomerType`: B2B 행에만 — 병원 / 병원(프랜차이즈) / 병원(대리점) / 피부관리실 / 피부관리실(프랜차이즈) / 피부관리실(대리점) / 대리점 / 기타
- `country`: 수출 행에만 — 거래처 사업형태에서 추출
- `brandHouse`: `자체` (레노덤·레노덤프로·헤이우) | `수입` (네오·엑스비·크리스티나) | `기타`
- `channelGroup`: `자사 공식몰` | `종합몰` | `소호몰` | `면세점` | `B2B` | `수출` | `임직원/패밀리` | `기타`
- `gp`(이익) = `실매출 - 원가` (원가 NaN인 행은 NaN — 원가는 행당 합계임)
- `isNonRevenue`: 실매출 0 또는 사업형태가 증정/임직원/마케팅용/파손/테스트인 경우 true
- `yearMonth`: `YYYY-MM`

### 2.4 팩트 큐브 (v3, `lib/facts.ts`)

CSV 로드와 동시에 **모든 차원 사전 집계** 빌드. 페이지가 raw rows에서 매번 그룹화하지 않고 사전 셀을 룩업/합산.

**셀 (`FactCell`)**: `{ revenue, qty, orders:Set<주문번호>, discount, fee, shippingFee, settlement, orderAmount, gpSum, gpRevenueBase, costMissingCount, rowCount }`. `cellToKpi(cell)` 로 KPI 객체 변환.

**1D 인덱스** (월별 → 차원값 → 셀):
`byMonth` / `byMonthCategory` / `byMonthChannelGroup` / `byMonthChannel` / `byMonthBrand` / `byMonthBrandHouse` / `byMonthCustomer` / `byMonthDealer` / `byMonthCountry` / `byMonthB2bType`

**2D 인덱스**:
`byMonthDealerType` (영업사원 × 거래처유형) / `byMonthBrandChannelGroup` (B2C: 브랜드 × 채널그룹) / `byMonthCountryBrand` (수출: 국가 × 브랜드)

**기타**: `byMonthDealerCustomers` (딜러 → 거래처 Set, 활성 거래처 수 산출) / `byMonthProduct` (제품) / `byMonthNonRevBizType` (비매출) / `byMonthDay` (일별)

**메타**: `monthsAsc` / `customers/dealers/brands/channels/countries` Set / `customerToCategory|Brand|Dealer` (전체 기간 매출 최대 기준 대표값)

**규약**: 신규 분석/인사이트 코드는 `loadFactCube()` 직접 사용. 큐브에 인덱스가 없는 분해(예: 거래처×제품)는 한 달치 raw 한 번 스캔으로 처리. 전체 raw 스캔 금지.

## 3. 분류 매핑 (사용자 정의 명시화)

매핑 파일(`config/mappings.ts`)에 **모두 상수**로 두고 매월 점검.

### 3.1 채널 → 대분류

- **수출**: `수출`
- **B2B**: `B2B몰`
- **면세점**: `면세점` *(B2C 안의 별도 그룹이지만 KPI는 독립)*
- **B2C**: 위 셋을 제외한 모든 채널 (16종)

### 3.2 B2C 내 채널 그룹

| 그룹 | 채널 |
|---|---|
| 자사 공식몰 | 레노덤 공식몰, 레노덤 스마트스토어, 엑스비앙스 공식몰, 엑스비앙스 스마트스토어, 헤이우 공식몰, 바크로하우스, 바크로하우스 스마트스토어 |
| 종합몰 | W컨셉, SSG, 쿠팡, 쿠팡 로켓, 쿠팡 그로스, 큐텐, 화해 |
| 소호몰 | 소호몰 |
| 임직원/패밀리 | 바크로패밀리, 헤메코랩 |

### 3.3 브랜드 → 하우스

- **자체**: 레노덤, 레노덤 프로페셔널, 헤이우
- **수입**: 네오스트라타, 엑스비앙스, 크리스티나
- **기타**: 기타 *(증정/리플렛/도구류 — 매출 집계에서 보통 0)*

### 3.4 브랜드 ↔ 공식몰 매핑

- 레노덤: `레노덤 공식몰` + `레노덤 스마트스토어`
- 엑스비앙스: `엑스비앙스 공식몰` + `엑스비앙스 스마트스토어`
- 헤이우: `헤이우 공식몰`
- 바크로하우스(다브랜드): `바크로하우스` + `바크로하우스 스마트스토어`

### 3.5 수출 국가 (거래처 사업형태에서 추출)

`해외 (XXX)` 또는 `수출 (XXX)` 패턴에서 괄호 안만 추출. 미매칭(`해외 (기타)`, `수출 (기타)`)은 **"기타"**.

### 3.6 비매출 출고 (사업형태)

증정 (기타), 증정 (마케팅), 증정 (영업), 임직원, 직원, 거래처 직원, 마케팅용, 테스트 (수입허가), 파손제품, 교육 → **비매출 출고**로 묶어 별도 노출.

## 5. 페이지별 상세 기획

### Tab 1. 종합 — "이번 달, 한 페이지로"

**0) 자동 인사이트** — 전월/전년 변화, 카테고리 빅 무버, 거래처 상위 변동, 동면 복귀, 분기 절벽, 상실된 핵심

**1) 헤더 KPI 4개 카드** — 모두 MoM·QoQ·YoY ▲▼% 동시 표기
- 전체 실매출 (목표 달성률 포함)
- 수출 실매출
- 국내 실매출 (B2C+B2B+면세점)
- 총 판매수량

**2) 변화 요인 워터폴** — 카테고리 단위 + 거래처 단위 (Top/하위 5)

**3) 일별 누적 매출 라인** — 이번달/전월/전년 동월 3선

**4) 12개월 카테고리 스택바 + 카테고리 도넛**

**5) Top 10 거래처 / Top 10 제품** — 2개 테이블 (실매출/수량/MoM%)

**6) 거래처 변동 하이라이트 (`<AccountHighlights>`)** *(v3 신규)*
- 전월 대비 상위 상승 / 하락 (각 5)
- 동면 거래처 복귀 (직전 3개월 무거래 → 이번달 매출, 300만원 이상)
- 분기 절벽 경보 (지난 분기 상위 30 중 진행률 보정 -40%↓)
- 상실된 핵심 거래처 (지난 분기 상위 10 → 이번달 0원)
- 모든 거래처명은 클릭 시 `/accounts?customer=XXX&month=YYYY-MM` 이동

**7) 비매출 출고 카드** — 증정/임직원/마케팅용/파손 등으로 나간 "매출 0" 출고 (수량·원가 합)

## 6. KPI/계산식 정의

| 지표 | 정의 |
|---|---|
| 실매출 | `실 매출` 컬럼 합 |
| 주문건수 | unique `주문번호` |
| AOV (객단가) | 실매출 / 주문건수 |
| 수량 | `판매수량` 합 |
| 정산매출 | `정산금액` 합 |
| 수수료율 | `수수료` / `주문금액` |
| 할인율 | `할인금액` / `주문금액` |
| GP (매출총이익) | `실매출 - 원가` (원가 NaN 행 제외, 원가는 행당 합계) |
| GP율 | GP / 해당 부분집합 실매출 (원가 있는 행만) |
| 전월(MoM) | (당월-전월) / 전월 |
| 전분기(QoQ) | (당분기 누적-직전분기 누적) / 직전분기 누적 |
| 전년 동월(YoY) | (당월-전년동월) / 전년동월 |
| 분기 절벽 | (이번 분기 누적 - 지난 분기 합 × (진행 개월/3)) / (지난 분기 합 × (진행 개월/3)) ≤ -0.4 |
| 동면 복귀 | 직전 3개월 모두 0원 + 이번달 매출 ≥ 100만원 + 직전 12개월 lookback |
| 거래처 집중지수(HHI) | Σ (거래처 매출 / 전체 매출)² × 10000 |

분기 진행 중일 땐 카드 위에 "분기 진행 중 (N/3개월)" 표시.

## 7. 자동 인사이트 휴리스틱 (`lib/tabInsights.ts`)

LLM 사용 금지. 결정적 규칙으로 노이즈 없는 한국어 불릿 5~7개 생성.

### 공통 임계치

| 항목 | 임계치 |
|---|---|
| 전체 매출 변화 | ±3% 이상 |
| 차원별 빅 무버 | 절대 차액 ≥ 500만원 + ±15% 이상 |
| 신규/이탈 (큰 규모) | 전월 또는 이번달 매출 ≥ 500만원 |
| 동면 복귀 (종합 노출) | ≥ 300만원 |
| 목표 달성 | < 80% (미달 경고) 또는 > 110% (초과) |

### Severity 우선순위

`critical` (이탈/대폭 하락) > `warn` (-15%~-30% 또는 미달) > `positive` (성장/초과) > `info` (정보성)

동률 시 절대 차액 큰 순.

### 탭별 특화

- **종합**: 전월/전년 변화 + 카테고리 빅 무버 (Top 2) + 거래처 변동 (상승 1, 하락 1) + 동면 복귀 1 + 분기 절벽 1 + 상실된 핵심 1
- **B2C**: 채널그룹 (자사몰/종합몰/소호몰) + 채널 빅 무버 (네이버 스마트스토어 등 — B2B몰/수출/면세점 제외) + 브랜드 빅 무버
- **B2B**: 영업사원 빅 무버 + 거래처유형 + 신규/이탈 카운트 + 신규 큰 거래처
- **수출**: 국가 빅 무버 + 신규/이탈 국가
- **면세점**: 면세점 거래처 빅 무버 (롯데/신라/신세계)
- **브랜드 분석**: 24m 신기록/최저 + 신규 SKU 수
- **거래처 분석**: 거래처 미선택 시 — Top 변동 / 핵심 이탈 / 신규 진입 요약. 선택 시 — 그 거래처의 전월/전년 변화 + 플래그
- **목표 달성**: 전체 달성률 + Top 미달 3 + Top 초과 2 + 신규 추진 진척
- **변동 분석**: 전월 변화 + 거래처 Top 2 상승/하락

## 8. 비교 로직과 시각 규약

- 양수 = **녹색 ▲**, 음수 = **빨강 ▼**, ±2% 이내 = **회색 ●**
- 전년 동월(YoY) 데이터가 없는 항목은 "신규/N.A."로 표기
- 차트의 점선/그림자는 항상 "전년 동월" 의미로 통일
- 모든 비교 카드: **이번달 절대값 + 비교 라벨 + 비교 절대값 + 차이금액 + 변화율** 동시 노출
- 통화 단위: `formatKRW` (정확값) / `formatKRWLong` (1억 5,623만원) / `formatKRWShort` (좁은 공간) — 영문 단위(M/B) 절대 금지

## 9. 매월 운영 워크플로우

1. **데이터 교체**: `sales.csv` (필요 시 `target.csv`)를 최신 파일로 덮어쓰기
2. **자동 검증**: `npm run check` → 매핑 누락 채널/브랜드/사업형태 경고
3. **(선택) 사람 코멘트 추가**: `insights/2026-05.md` 생성, 자동 분석으로 잡히지 않는 맥락만 기록
4. **보고서 출력**: 브라우저에서 `?month=2026-05` → 인쇄 또는 `npm run snapshot -- 2026-05`

새 채널/거래처가 등장해도 코드 수정 없이 자동 분류:
- 채널: 화이트리스트 매핑 → 미매칭은 자동으로 "기타"로 떨어지되 경고
- 거래처: 패턴(`해외 (XXX)`, `병원`, `피부관리실`) 우선

## 11. 정책 결정 (사용자 확정)

- **임직원/증정/마케팅용**: 매출에서 제외하되 **"비매출 출고" 별도 카드/탭으로 노출**
- **수입 브랜드 (네오·엑스비·크리스티나)**: 데이터 그대로 노출 (특수 캡션 없음)
- **자동 인사이트가 기본**: 모든 탭 상단에 휴리스틱 자동 인사이트. 사람 코멘트는 **선택** — `insights/YYYY-MM.md` 가 있을 때만 `/insights` 페이지 하단 마크다운 렌더
- **거래처/딜러 분석 깊이**: 단순 Top-N으로 끝내지 말 것. 동면 복귀 / 분기 절벽 / 상실된 핵심 / 신규 진입 자동 감지
- **거래처 클릭 통일**: 어디서든 거래처명 클릭 → `/accounts?customer=XXX&month=YYYY-MM` 이동
- **신규 분석은 큐브 직접 사용**: raw rows 전체 스캔 금지
